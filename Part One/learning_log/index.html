My next task was to analyse this script and break down what is happening statement by statement. This I will do below:

  1. Upon function invocation, the document checks whether there is a value for phone number. If there is no value, the user is alerted and the method returns false and the cursor is focused on the phone text element
  2. If there is a value, empty variables are declared to store the numbers and characters, and a variable phoneNo is declared to take the value of the form input
  3. Next, it iterates through every character in phoneNo and if the character is a value between 0 and 9, i.e. numeric, it gets added to the variable numbersOnly
  4. Following this, numbersOnly should be a string composed of numeric values. Next is a check to see if it is 13 characters long. If not, an error is thrown and the user is alerted and the function returns false. The cursor is focused on the phone text element.
  5. Now we have a 13 character string composed of numbers only. We make use of substring functions to break this phone number down into its components. These are the 2-digit area code, followed by the leading 0, 2-digit exchange, 4-digit extension 1 and 4-digit extension 2.
  6. A new number is then created which formats the phone in the form +## (#)## ####-####
  7. The script then sets this newNumber to be the value in the phone text element. Due to how HTML handles form submissions, this is immediately cleared upon clicking submit so I added a console log at the bottom to confirm it works.

Following this, I implmeneted the script in my previous HTML document so instead of checking for valid names, it checked the validNo method described above. Task 5 was to explain the workings of this validateEmail function. This is as below:

  1. Upon function invocation, the document checks whether there is a value for eMail. If there is no value, the user is alerted and the method returns false and the cursor is focused on the eMail text element
  2. If there is a value for email address, variable atLoc is declared to be the index of the '@' symbol in the email string, as long as it is not the first character.
  3. Variable dotLoc is declared to be the index of the '.' character, as long as that character's position is at least 2 characters greater than that of atLoc
  4. Variable len is declared to be the length of the email address
  5. Email address is considered to be valid if atLoc and dotLoc are greater than 0, and if len is greater than dotLoc+2. i.e. the length of the email is greater than dotLoc with one appended character. In other words, at least 2 characters must exist after the dot.
  6. If this is false, the user is alerted that the email is invalid and told to enter again. The document focuses on the email text element and the method returns false.
